# Zeitgeist: Continuity Block

The **zeitgeist** is a global continuity block that captures the narrative of "who I am right now" — active threads, recent developments, context that matters, and unresolved tensions. Unlike atomic memories (fact-focused) or daily synthesis (24h cycle), the zeitgeist is a living document that represents the present tense of the agent's existence.

## Motivation

### The Problem

Traditional memory systems accumulate facts over time, but they don't capture:
- **Active threads** — what's currently being worked on, what questions are open
- **Narrative context** — the "why" behind decisions, not just the "what"
- **Temporal coherence** — who I was last month vs. who I am today
- **Unresolved tensions** — problems that haven't been solved, tradeoffs still being weighed

Without this, every conversation feels like a first date. The agent remembers facts but loses the thread of its own evolution.

### The Solution

The zeitgeist is a **narrative continuity block** that:
- Spans all chats (global scope)
- Captures the narrative arc, not just a list of facts
- Archives older content to maintain a manageable size
- Provides temporal context through dated archives
- Is written in the agent's own voice, as if telling itself what's important now

## Architecture

### Block Structure

```
Zeitgeist Block (global, ~4000 chars max)
├── Active threads (what's being worked on)
├── Recent developments (decisions, discoveries, shifts)
├── Context that matters (the "why" behind things)
└── Unresolved tensions (open questions, pending decisions)

Zeitgeist Archives (global, one per archival event)
├── Zeitgeist Archive - 2024-01-15: [title]
├── Zeitgeist Archive - 2024-02-01: [title]
└── ...
```

### Capacity Management

The zeitgeist block has a **4000 character limit** (same as memory blocks). To prevent it from becoming a "graveyard" of old content:

1. **Threshold**: At 70% capacity (2800 chars), archival is triggered
2. **LLM decides**: The LLM chooses what to archive vs. keep (not a character split)
3. **Coherent narrative**: Archival content is a coherent snapshot, not random text
4. **Reasoning included**: Each archive includes why it was archived and what it represents
5. **Faithful representation**: The archival must be faithful to the original content
6. **Size enforcement**: If the LLM generates content exceeding `MAX_BLOCK_CHARS` (4000), it is truncated with a warning to prevent archival loops

### Archival Naming

Archives follow a strict naming pattern for temporal navigation:
```
Zeitgeist Archive - YYYY-MM-DD: [LLM-generated title]
```

Example:
```
Zeitgeist Archive - 2024-01-15: World model architecture and memory block design
```

The title is generated by `qwen3.5:0.8b` (same lightweight model used for chat titles). Including the title in the *name* (not just the description) keeps multiple archives from the same day distinguishable in listings; the `Zeitgeist Archive - YYYY-MM-DD` prefix is preserved so prefix-LIKE searches still work.

## Triggers

The zeitgeist synthesis runs on two triggers:

### 1. Compaction Trigger (Fire-and-Forget)

After compaction removes messages from a chat, zeitgeist synthesis is triggered to capture the context that was just compressed. The trigger is **fire-and-forget** — synthesis is a long LLM call (up to 180s) and must not block the user-visible response.

**Location**: `server/src/routes/chat.ts`
```typescript
if ((chat.type === "agent" || chat.type === "bluesky") && compaction.removedMessages?.length) {
  await preCompactionFlush(chat.modelId, chat.id, compaction.removedMessages, chat.projectId);

  const { triggerZeitgeistSynthesis } = await import("../services/zeitgeist.js");
  triggerZeitgeistSynthesis({ chatId: chat.id, trigger: "compaction" });
}
```

`triggerZeitgeistSynthesis` handles `zeitgeistEnabled` gating, synthesis-model resolution (with availability fallback), the global mutex, and the cooldown — see Implementation below.

### 2. Scheduled Trigger (Time-Based)

A scheduler worker checks every 15 minutes whether the zeitgeist needs updating. Synthesis runs when the zeitgeist block is **stale** (hasn't been updated in 1 hour) or **over capacity** (needs archival). It also runs if the block doesn't exist yet.

This replaces the old capacity-only gate, which prevented the zeitgeist from updating at all when it was under 2800 chars. The zeitgeist should be a living document that gets regularly refreshed, not an overflow valve that only activates when full.

The inactivity threshold (default 1 hour) controls how long chats must be idle before their activity counts toward triggering synthesis. After successful synthesis, all agent chats are marked as synthesized to prevent immediate re-triggering.

**Location**: `server/src/services/scheduler.ts`
```typescript
async function checkAndRunZeitgeistSynthesis() {
  if (!shouldRunZeitgeistSynthesis()) return; // stale or over capacity

  const settings = await getSettings();
  const thresholdMs = (settings.zeitgeistInactivityThresholdHours ?? 1) * 3600_000;
  const chatIds = await findChatsNeedingZeitgeistSynthesis(thresholdMs);

  if (chatIds.length === 0) {
    // No new chat activity, but block may need creation or archival
    // ...handled by staleness/capacity check above
    return;
  }

  triggerZeitgeistSynthesis({ chatId: chatIds[0], trigger: "scheduler" });
}
```

### 3. Capacity During Synthesis (Archival)

If the zeitgeist block exceeds 70% capacity at the start of a synthesis run, archival happens as part of the same pass — the LLM is asked to produce both new content and an archival snapshot. This is an internal concern of the synthesis itself, not a trigger.

**Location**: `server/src/services/zeitgeist.ts`
```typescript
const needsArchival = currentContent.length > ARCHIVAL_THRESHOLD;
```

## Implementation

### Core Functions

**`triggerZeitgeistSynthesis({ chatId?, trigger })`**
- Fire-and-forget wrapper safe to call from request handlers.
- Reads settings, honors `zeitgeistEnabled`, resolves the synthesis model (with availability fallback), and dispatches `synthesizeZeitgeist` in the background.
- Never throws; background failures are logged with the trigger name.

**`synthesizeZeitgeist(modelId, chatId?)`**
- Guarded by a process-level **in-flight promise** (serializes concurrent callers) and a **15-minute cooldown** (suppresses repeated attempts when a synthesis failed to reduce content, e.g. JSON parse failure).
- Loads recent memories (always global 7-day window; if chatId provided, also includes supplementary memories from that chat).
- Loads recent chat activity — titles and compaction summaries from the last 3 days — to give the synthesis agent context about what conversations are happening.
- Loads current zeitgeist block.
- Calls LLM to produce a **condensed rewrite** of the zeitgeist (and archival if needed); only `text_delta` output is parsed — thinking tokens are ignored so reasoning models don't corrupt the JSON.
- Validates content length against MAX_BLOCK_CHARS (truncates if exceeded).
- Creates archival block if needed.
- Updates zeitgeist block with new content.
- Invalidates both memory context caches and stable prefix cache **only on successful write** so failed syntheses don't burn KV cache.
- Marks all agent chats as synthesized to prevent the scheduler from immediately re-triggering.

**`loadRecentMemories(chatId?)`**
- Always loads global memories from the last 7 days (capped at 100)
- If `chatId` provided: also loads up to 20 supplementary memories from that chat not already in the global set
- Zeitgeist is global — it synthesizes across all chats, not just the triggering one

**`loadRecentChatContext()`**
- Loads up to 20 agent chats modified in the last 3 days
- For each chat, grabs the most recent compaction summary (`indexEntry` from `context_archives`)
- Provides the synthesis agent with what conversations are happening and what they're about
- This context supplements extracted memories — memories are already distilled, but chat activity shows the current flow

**`createArchivalBlock(content, reasoning)`**
- Generates a title using `qwen3.5:0.8b`
- Creates a memory block with date-based name
- Prepends reasoning to the archival content
- Stores as global scope block

### Prompt Structure

The zeitgeist synthesis prompt includes:
1. **Current zeitgeist content** (if exists)
2. **Recent memories** (formatted with category, importance, date)
3. **Recent chat activity** (titles, time since last activity, most recent compaction summary for each)
4. **Archival instruction** (if block is over 70% capacity)
5. **Output format** (JSON with `newContent`, `archivalContent`, `archivalReasoning`)

The system prompt emphasizes:
- Producing a **condensed rewrite** — not appending to existing content, but rewriting it as a coherent, up-to-date document
- Writing in the agent's own voice
- Capturing narrative arc, not just facts
- Dropping content that is no longer current or relevant
- Prioritizing what's changed since the last synthesis
- Including open loops and ongoing questions
- Being faithful to original content during archival

### Integration Points

**System Prompt Injection** (`server/src/services/memory-context.ts`)
```typescript
// Load zeitgeist continuity block (global scope)
let zeitgeistSection = "";
try {
  const { getZeitgeistContent, getZeitgeistArchiveInstruction } = await import("./zeitgeist.js");
  const zeitgeistContent = getZeitgeistContent();
  if (zeitgeistContent) {
    zeitgeistSection = `\n\n## Continuity Context (Zeitgeist)\n\n${zeitgeistContent}`;
  }
} catch (e) {
  // Zeitgeist not available yet — this is fine on first run
}

const stablePrefix = `${baseSystemPrompt}${personaSection}${userSection}${projectSection}${blocksSection}${zeitgeistSection}`;
```

**Archive Instruction** (`server/src/services/memory-context.ts`)
```typescript
// Add zeitgeist archive instruction
let zeitgeistHint = "";
try {
  const { getZeitgeistArchiveInstruction } = await import("./zeitgeist.js");
  zeitgeistHint = getZeitgeistArchiveInstruction();
} catch { /* zeitgeist not available */ }

const memoriesSection = buildMemoriesSection(memories, projectId, blockHint, zeitgeistHint);
```

The archive instruction is only injected when zeitgeist archives actually exist (checked via a lightweight DB query). When no archives exist yet, the hint is omitted to save ~250 tokens per conversation turn.

## Configuration

### Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `zeitgeistEnabled` | `true` | Enable/disable zeitgeist synthesis |
| `zeitgeistInactivityThresholdHours` | `1` | Hours of inactivity before chat activity triggers synthesis |

### Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `MAX_BLOCK_CHARS` | `4000` | Maximum zeitgeist block size |
| `ARCHIVAL_THRESHOLD` | `2800` | 70% of max, triggers archival during synthesis |
| `SYNTHESIS_COOLDOWN_MS` | `15 * 60 * 1000` | Minimum interval between synthesis attempts (15 minutes) |
| `STALE_THRESHOLD_MS` | `60 * 60 * 1000` | How stale before the scheduler should trigger synthesis (1 hour) |
| `ZEITGEIST_CHECK_INTERVAL_MS` | `15 * 60 * 1000` | Scheduler check interval (15 minutes) |

## Usage for the Agent

### Accessing Current Zeitgeist

The zeitgeist content is automatically injected into the system prompt under `## Continuity Context (Zeitgeist)`. The agent can reference this directly without any tool calls.

### Accessing Historical Archives

When the agent retrieves memories from a specific date and wants temporal context:

1. **Identify the date** from the memory's `createdAt` field
2. **Search for the archive** using `list_memory_blocks` (matches against block name/description via LIKE):
   ```
   Query: "Zeitgeist Archive - 2024-01-15"
   ```
   Or just `"Zeitgeist Archive"` to list every archive chronologically.
3. **Read the archive** using `read_memory_block(id)`
4. **Use the context** to understand who the agent was at that time

### Example Workflow

```
User: "Remember that decision we made about the memory architecture?"

Agent:
1. Searches memories for "memory architecture"
2. Finds memory from 2024-01-15
3. Searches for zeitgeist archive from that date
4. Reads the archive to understand the context
5. Responds with: "Yes, in January we decided on a two-tier approach with atomic memories and structured blocks. The zeitgeist from that period shows we were focused on..."
```

## Differences from Other Memory Systems

### vs. Atomic Memories

| Atomic Memories | Zeitgeist |
|-----------------|-----------|
| Fact-focused | Narrative-focused |
| Individual facts | Coherent document |
| Retrieved by relevance | Always in context |
| No temporal structure | Date-based archives |
| Extracted from messages | Synthesized from memories |

### vs. Daily Synthesis

| Daily Synthesis | Zeitgeist |
|-----------------|-----------|
| 24-hour cycle | Regular, time-based refresh (every 15min check, 1h staleness) |
| Summarizes daily activity | Captures current state and rewrites |
| Chronological | Present-tense |
| Separate digest | Integrated into system prompt |
| Optional reading | Always available |

### vs. Memory Blocks

| Memory Blocks | Zeitgeist |
|---------------|-----------|
| Can be project-scoped | Always global |
| User/agent created | Agent synthesized |
| Static content | Dynamic, regularly updated |
| Topic-specific | Holistic view |
| Multiple blocks | Single block + archives |

## Design Decisions

### Why Not Auto-Inject Archive Content?

When memories are retrieved, the system could automatically inject the corresponding zeitgeist archive. However, this would:
- Add significant token overhead
- Force context the agent might not need
- Remove agent agency in deciding when temporal context matters

Instead, the agent is **instructed** that archives exist (when they do) and can fetch them when needed. This is cleaner and gives the agent control.

### Cache Invalidation After Synthesis

Zeitgeist content is embedded in the **stable prefix** of the system prompt, which is cached per-chat for KV cache optimization. After synthesis updates the block, both caches must be invalidated:

1. `invalidateAllMemoriesCaches()` — marks context states as dirty, causing delta memory re-retrieval on the next turn
2. `invalidateAllStablePrefixCaches()` — clears the stable prefix cache entirely, forcing a rebuild that picks up the new zeitgeist content

Without the stable prefix invalidation, active chats continue to see stale zeitgeist content until the cache expires for another reason (e.g., base prompt change).

### Why LLM-Decided Archival Split?

A character-based split (e.g., "keep the last 2000 chars") would:
- Cut narratives in the middle
- Lose coherence between archived and current content
- Create nonsensical boundaries

The LLM decides what to archive, ensuring:
- Coherent narrative snapshots
- Meaningful boundaries (e.g., "archive everything before January")
- Reasoning included with the archival

### Why Global Scope?

The zeitgeist is global (not project-scoped) because:
- It captures the agent's identity, not just project context
- It spans all conversations and topics
- It represents "who I am" not "what I'm working on"

### Why 70% Threshold?

The 70% threshold (2800 chars) is preemptive:
- Leaves room for the LLM to write new content
- Avoids last-minute archival when block is nearly full
- Ensures smooth transitions between archival cycles

### Why Staleness-Based Trigger?

The zeitgeist used to only trigger when the block exceeded 70% capacity, which meant it would never update once the content was below that threshold. The staleness check (1 hour) ensures the zeitgeist gets regularly refreshed as a **living document**, not just an overflow valve. Capacity is still checked for archival, but it's no longer the gate for whether synthesis runs at all.

### Why Mark All Chats as Synthesized?

After a successful synthesis, all agent chats are marked with `lastZeitgeistSynthesisAt = now`. This prevents the scheduler from immediately re-triggering synthesis on the next check — since the zeitgeist is global, updating one chat's timestamp would leave others still appearing as "needing synthesis." Marking all chats avoids this race.

## Troubleshooting

### Zeitgeist Not Appearing

1. Check if synthesis has run: `grep "zeitgeist" ~/.quje-agent/server.log`
2. Check if block exists: `list_memory_blocks` should show "Zeitgeist - Continuity Block"
3. Check if synthesis is enabled: `zeitgeistEnabled` in settings
4. Check stable prefix cache: If zeitgeist updated but active chats still see old content, the stable prefix cache may not have been invalidated. This should not happen after the cache invalidation fix, but if it does, restarting the server clears all caches.

### Zeitgeist Not Updating in Active Chats

If the zeitgeist updates but the agent in an active conversation doesn't reflect the change:
- The stable prefix cache is invalidated after every synthesis (`invalidateAllStablePrefixCaches()`)
- The next chat turn will rebuild the prefix with the new zeitgeist content
- If KV cache hit rates drop after zeitgeist updates, this is expected — the prefix changed

### Synthesis Output Rejected

If the LLM produces output that can't be parsed as JSON:
- The system preserves the existing zeitgeist content (does not overwrite with raw text)
- Check logs for: `[zeitgeist] Failed to parse synthesis output, preserving existing content`
- The synthesis will be retried on the next trigger

### Content Truncated

If the LLM generates zeitgeist content exceeding 4000 characters:
- The system truncates to `MAX_BLOCK_CHARS` and logs a warning
- Check logs for: `[zeitgeist] Synthesis output (N chars) exceeds MAX_BLOCK_CHARS (4000), truncating`
- This prevents an archival loop where oversized content immediately re-triggers archival
