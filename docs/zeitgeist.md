# Zeitgeist: Continuity Block

The **zeitgeist** is a global continuity block that captures the narrative of "who I am right now" — active threads, recent developments, context that matters, and unresolved tensions. Unlike atomic memories (fact-focused) or daily synthesis (24h cycle), the zeitgeist is a living document that represents the present tense of the agent's existence.

## Motivation

### The Problem

Traditional memory systems accumulate facts over time, but they don't capture:
- **Active threads** — what's currently being worked on, what questions are open
- **Narrative context** — the "why" behind decisions, not just the "what"
- **Temporal coherence** — who I was last month vs. who I am today
- **Unresolved tensions** — problems that haven't been solved, tradeoffs still being weighed

Without this, every conversation feels like a first date. The agent remembers facts but loses the thread of its own evolution.

### The Solution

The zeitgeist is a **narrative continuity block** that:
- Spans all chats (global scope)
- Captures the narrative arc, not just a list of facts
- Archives older content to maintain a manageable size
- Provides temporal context through dated archives
- Is written in the agent's own voice, as if telling itself what's important now

## Architecture

### Block Structure

```
Zeitgeist Block (global, ~4000 chars max)
├── Active threads (what's being worked on)
├── Recent developments (decisions, discoveries, shifts)
├── Context that matters (the "why" behind things)
└── Unresolved tensions (open questions, pending decisions)

Zeitgeist Archives (global, one per archival event)
├── Zeitgeist Archive - 2024-01-15: [title]
├── Zeitgeist Archive - 2024-02-01: [title]
└── ...
```

### Capacity Management

The zeitgeist block has a **4000 character limit** (same as memory blocks). To prevent it from becoming a "graveyard" of old content:

1. **Threshold**: At 70% capacity (2800 chars), archival is triggered
2. **LLM decides**: The LLM chooses what to archive vs. keep (not a character split)
3. **Coherent narrative**: Archival content is a coherent snapshot, not random text
4. **Reasoning included**: Each archive includes why it was archived and what it represents
5. **Faithful representation**: The archival must be faithful to the original content

### Archival Naming

Archives follow a strict naming pattern for temporal navigation:
```
Zeitgeist Archive - YYYY-MM-DD: [LLM-generated title]
```

Example:
```
Zeitgeist Archive - 2024-01-15: World model architecture and memory block design
```

The title is generated by `qwen3.5:0.8b` (same lightweight model used for chat titles).

## Triggers

The zeitgeist synthesis runs on three triggers:

### 1. Compaction Trigger (Immediate)

After compaction removes messages from a chat, zeitgeist synthesis runs immediately to capture the context that was just compressed.

**Location**: `server/src/routes/chat.ts`
```typescript
if ((chat.type === "agent" || chat.type === "bluesky") && compaction.removedMessages?.length) {
  await preCompactionFlush(chat.modelId, chat.id, compaction.removedMessages, chat.projectId);
  
  // Trigger zeitgeist synthesis after compaction
  const { synthesizeZeitgeist } = await import("../services/zeitgeist.js");
  await synthesizeZeitgeist(chat.modelId, chat.id, false);
}
```

### 2. Deferred Trigger (Inactivity-Based)

A scheduler worker checks for inactive chats every 30 minutes. If a chat has been inactive for 4 hours (configurable) and the zeitgeist needs synthesis (capacity threshold), it triggers synthesis.

**Location**: `server/src/services/scheduler.ts`
```typescript
async function checkAndRunZeitgeistSynthesis() {
  const settings = await getSettings();
  const enabled = settings.zeitgeistEnabled ?? true;
  const thresholdHours = settings.zeitgeistInactivityThresholdHours ?? 4;
  
  if (!enabled) return;
  
  // Check if zeitgeist needs synthesis based on capacity
  if (!shouldTriggerZeitgeistSynthesis()) {
    console.log("[scheduler] Zeitgeist under capacity threshold, skipping deferred check");
    return;
  }
  
  // Find inactive chats and trigger synthesis
  const chatIds = await findChatsNeedingZeitgeistSynthesis(thresholdMs);
  if (chatIds.length === 0) return;
  
  await synthesizeZeitgeist(synthesisModelId, chatIds[0], false);
}
```

### 3. Capacity Trigger (Preemptive)

If the zeitgeist block exceeds 70% capacity during synthesis, archival is forced regardless of other conditions.

**Location**: `server/src/services/zeitgeist.ts`
```typescript
const needsArchival = forceArchive || currentContent.length > ARCHIVAL_THRESHOLD;
```

## Implementation

### Core Functions

**`synthesizeZeitgeist(modelId, chatId?, forceArchive?)`**
- Loads recent memories (last 7 days, or from specific chat)
- Loads current zeitgeist block
- Calls LLM to synthesize new content (and archival if needed)
- Creates archival block if needed
- Updates zeitgeist block with new content
- Invalidates memory caches so active chats see the update

**`loadRecentMemories(chatId?)`**
- If `chatId` provided: loads memories from that chat
- Otherwise: loads all memories from the last 7 days
- Caps at 100 memories to avoid context overflow

**`createArchivalBlock(content, reasoning)`**
- Generates a title using `qwen3.5:0.8b`
- Creates a memory block with date-based name
- Prepends reasoning to the archival content
- Stores as global scope block

### Prompt Structure

The zeitgeist synthesis prompt includes:
1. **Current zeitgeist content** (if exists)
2. **Recent memories** (formatted with category, importance, date)
3. **Archival instruction** (if needed)
4. **Output format** (JSON with `newContent`, `archivalContent`, `archivalReasoning`)

The system prompt emphasizes:
- Writing in the agent's own voice
- Capturing narrative arc, not just facts
- Including open loops and ongoing questions
- Being faithful to original content during archival

The archive instruction tells the agent:
```
## Temporal Context Access

Zeitgeist archives are available for historical context. Each archive represents a snapshot of the continuity block from a specific date. When you retrieve memories from a particular date, you can search for the corresponding zeitgeist archive using:

- Search query: "zeitgeist-archive-YYYY-MM-DD" (replace with the date)
- Use memory block search tools to retrieve the full archive content

This allows you to understand the narrative context from that period, not just isolated facts.
```

## Configuration

### Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `zeitgeistEnabled` | `true` | Enable/disable zeitgeist synthesis |
| `zeitgeistInactivityThresholdHours` | `4` | Hours of inactivity before deferred trigger |

### Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `MAX_BLOCK_CHARS` | `4000` | Maximum zeitgeist block size |
| `ARCHIVAL_THRESHOLD` | `2800` | 70% of max, triggers preemptive archival |
| `ZEITGEIST_CHECK_INTERVAL_MS` | `30 * 60 * 1000` | Scheduler check interval (30 minutes) |

## Usage for the Agent

### Accessing Current Zeitgeist

The zeitgeist content is automatically injected into the system prompt under `## Continuity Context (Zeitgeist)`. The agent can reference this directly without any tool calls.

### Accessing Historical Archives

When the agent retrieves memories from a specific date and wants temporal context:

1. **Identify the date** from the memory's `createdAt` field
2. **Search for the archive** using `list_memory_blocks` or `search_memory_blocks`:
   ```
   Query: "zeitgeist-archive-2024-01-15"
   ```
3. **Read the archive** using `read_memory_block(id)`
4. **Use the context** to understand who the agent was at that time

### Example Workflow

```
User: "Remember that decision we made about the memory architecture?"

Agent:
1. Searches memories for "memory architecture"
2. Finds memory from 2024-01-15
3. Searches for zeitgeist archive from that date
4. Reads the archive to understand the context
5. Responds with: "Yes, in January we decided on a two-tier approach with atomic memories and structured blocks. The zeitgeist from that period shows we were focused on..."
```

## Differences from Other Memory Systems

### vs. Atomic Memories

| Atomic Memories | Zeitgeist |
|-----------------|-----------|
| Fact-focused | Narrative-focused |
| Individual facts | Coherent document |
| Retrieved by relevance | Always in context |
| No temporal structure | Date-based archives |
| Extracted from messages | Synthesized from memories |

### vs. Daily Synthesis

| Daily Synthesis | Zeitgeist |
|-----------------|-----------|
| 24-hour cycle | Continuous, triggered by events |
| Summarizes daily activity | Captures current state |
| Chronological | Present-tense |
| Separate digest | Integrated into system prompt |
| Optional reading | Always available |

### vs. Memory Blocks

| Memory Blocks | Zeitgeist |
|---------------|-----------|
| Can be project-scoped | Always global |
| User/agent created | Agent synthesized |
| Static content | Dynamic, regularly updated |
| Topic-specific | Holistic view |
| Multiple blocks | Single block + archives |

## Design Decisions

### Why Not Auto-Inject Archive Content?

When memories are retrieved, the system could automatically inject the corresponding zeitgeist archive. However, this would:
- Add significant token overhead
- Force context the agent might not need
- Remove agent agency in deciding when temporal context matters

Instead, the agent is **instructed** that archives exist and can fetch them when needed. This is cleaner and gives the agent control.

### Why LLM-Decided Archival Split?

A character-based split (e.g., "keep the last 2000 chars") would:
- Cut narratives in the middle
- Lose coherence between archived and current content
- Create nonsensical boundaries

The LLM decides what to archive, ensuring:
- Coherent narrative snapshots
- Meaningful boundaries (e.g., "archive everything before January")
- Reasoning included with the archival

### Why Global Scope?

The zeitgeist is global (not project-scoped) because:
- It captures the agent's identity, not just project context
- It spans all conversations and topics
- It represents "who I am" not "what I'm working on"

### Why 70% Threshold?

The 70% threshold (2800 chars) is preemptive:
- Leaves room for the LLM to write new content
- Avoids last-minute archival when block is nearly full
- Ensures smooth transitions between archival cycles

## Troubleshooting

### Zeitgeist Not Appearing

1. Check if synthesis has run: `grep "zeitgeist" ~/.quje-agent/server.log`
2. Check if block exists: `list_memory_blocks` should show "Zeitgeist - Continuity Block"
3. Check if synthesis is enabled: `zeitgeistEnabled` in settings

### Archival Not Triggering

1. Check block size: Should be > 2800 chars to trigger
2. Check synthesis logs: Look for "needs archival" messages
3. Check LLM output: Should include `archivalContent` field

### Archives Not Found

1. Check naming pattern: Should be "Zeitgeist Archive - YYYY-MM-DD"
2. Check search query: Use exact date format
3. Check scope: Archives are global, not project-scoped
